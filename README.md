# 🐧 Linux 基础快速指北 (A Simple Guide to Linux)

> 这是一份为开发者准备的 Linux 基础入门指南，涵盖了核心概念与日常最高频使用的命令。

## 🌟 什么是 Linux？

Linux 是一套免费使用和自由传播的类 UNIX 操作系统。它不仅是一个内核（Kernel），更是一个庞大的开源生态。今天，全球绝大多数的服务器、云计算平台以及智能设备都在运行 Linux。

**Linux 的三大核心思想：**

1. **一切皆文件 (Everything is a file)：** 在 Linux 中，无论是硬件设备、网络套接字还是普通文本，统统被抽象为文件来管理。
2. **单一职责原则：** 每个程序或命令只做一件事，并且把它做好。通过管道符（`|`）将小工具组合起来完成复杂任务。
3. **完全由文本文件控制：** 系统的配置和管理几乎完全依赖于纯文本配置文件。

## 📂 核心目录结构 (FHS)

Linux 没有 Windows 那样的 C盘、D盘概念，一切都从根目录 `/` 开始呈树状分布：
| 目录路径 | 核心作用 | 常见场景 |
| :--- | :--- | :--- |
| `/` | 根目录 | 整个文件系统的起点 |
| `/home` | 用户主目录 | 存放普通用户的个人文件和配置 |
| `/etc` | 配置文件目录 | 存放系统和应用的核心配置文件 (如 nginx.conf) |
| `/var` | 可变数据目录 | 存放经常变化的文件，如**系统日志** (`/var/log`) |
| `/bin` | 可执行命令目录 | 存放所有用户都能执行的基础命令 (`ls`, `cp` 等) |
| `/root` | 超级管理员目录 | Root 用户的专属家目录 |

## 💻 开发者高频命令速查表 (Cheat Sheet)
### 1. 文件与目录操作
| 命令 | 说明 | 示例用法 |
| :--- | :--- | :--- |
| `pwd` | 显示当前所在路径 | `pwd` |
| `cd` | 切换目录 | `cd /var/log` (进入日志目录) |
| `ls` | 列出目录内容 | `ls -lh` (以人类可读格式显示详细信息) |
| `mkdir` | 创建新目录 | `mkdir my_project` |
| `rm` | 删除文件或目录 | `rm -rf tmp/` (危险：强制删除且不确认) |

### 2. 文本查看与处理
| 命令 | 说明 | 示例用法 |
| :--- | :--- | :--- |
| `cat` | 查看文件全部内容 | `cat config.yaml` |
| `tail` | 查看文件尾部内容 | `tail -f app.log` (**排障神器**：实时滚动查看日志) |
| `grep` | 文本过滤搜索 | `grep "ERROR" app.log` (找出包含报错的行) |

### 3. 系统状态与网络排障
| 命令 | 说明 | 示例用法 |
| :--- | :--- | :--- |
| `top` | 实时查看系统资源与进程 | `top` (按 `q` 退出) |
| `ps` | 查看当前进程状态 | `ps -ef \| grep python` (查找 Python 进程) |
| `ping` | 测试网络连通性 | `ping google.com` |
| `netstat` | 查看网络端口状态 | `netstat -tulnp` (查看当前监听的端口) |

## 🔒 权限管理基础

Linux 的权限分为三种角色：拥有者 (Owner)、所属组 (Group)、其他用户 (Others)。
权限类型分为：
* **r (Read, 4):** 读权限
* **w (Write, 2):** 写权限
* **x (Execute, 1):** 执行权限

> 示例：`chmod 755 script.sh` 
> 表示赋予拥有者读+写+执行权限 (4+2+1=7)，赋予同组和其他用户读+执行权限 (4+1=5)。
